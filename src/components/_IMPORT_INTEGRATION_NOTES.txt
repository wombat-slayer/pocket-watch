Sprint 5 Integration Notes
==========================

TransactionForm (existingTransactions prop)
-------------------------------------------
TransactionForm now accepts an optional `existingTransactions` prop (array of transaction objects).
When provided, handleSave will run a soft duplicate check before saving and show a window.confirm()
warning if a near-match is found (same amount ±$0.01, similar description, within 2 days).

To enable this for the Add Transaction modal in App.jsx, pass the prop:
  <TransactionForm
    accounts={accounts}
    existingTransactions={transactions}   // <-- add this line
    onSave={tx => { onAdd(tx); setShowAdd(false); }}
    onClose={() => setShowAdd(false)}
    userCategories={userCategories}
  />

The same prop can be passed to the Edit Transaction modal if desired. The duplicate check
automatically skips self-comparison (by id) so editing an existing transaction won't trigger
a false duplicate alert against itself.

CSVImport (no new props needed)
--------------------------------
CSVImport already receives `existingTxs` from Transactions.jsx. The improved duplicate
detection (findDuplicates) runs automatically after parsing and marks rows with `_isDuplicate`.
The `_isDuplicate` field is stripped before calling onImport, so App.jsx / the data store
are unaffected. No prop changes needed for CSVImport or Transactions.

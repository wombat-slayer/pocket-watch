ONBOARDING WIZARD INTEGRATION NOTES
=====================================
These changes need to be made in App.jsx to wire up OnboardingWizard.jsx.

1. IMPORT TO ADD (top of App.jsx with other component imports):
   import OnboardingWizard from './components/OnboardingWizard.jsx';

2. STATE TO ADD (inside the App component, with other useState declarations):
   const [showOnboarding, setShowOnboarding] = useState(false);

3. WHERE TO SET IT (inside the data-loading useEffect, after data is loaded):
   if (!data.onboardingComplete) setShowOnboarding(true);

4. HOW TO RENDER (inside the JSX return, e.g. just before the closing root div):
   {showOnboarding && (
     <OnboardingWizard
       onComplete={(accts, budgets) => {
         // Add accounts returned from wizard
         // accts is an array of account objects (may be empty if skipped)
         // budgets is an array of {category, amount} objects (may be empty if skipped)
         // Merge into app data and mark onboarding complete:
         setData(prev => ({
           ...prev,
           accounts: [...(prev.accounts || []), ...accts],
           budgets: [...(prev.budgets || []), ...budgets],
           onboardingComplete: true,
         }));
         setShowOnboarding(false);
       }}
     />
   )}

5. DATA FIELD TO PERSIST (ensure this field is included in your saved data object):
   onboardingComplete: true
   This field lives on the root data object. When false (or absent), the wizard
   shows on next launch. Once the user completes or skips the wizard, set it to
   true and save so it never appears again.

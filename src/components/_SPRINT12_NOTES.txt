TASK 3: Cleared Field Migration (App.jsx)
-----------------------------------------
STATUS: Already correct — no fix required.

migrateData() uses the pattern:
  {
    cleared: false,   // default comes BEFORE ...t spread
    ...t,             // existing transaction fields override defaults
  }

Any transaction that already has cleared: true in persisted storage will
have that value preserved. The migration correctly backfills cleared:false
only for old transactions that lack the field entirely. No bug present.

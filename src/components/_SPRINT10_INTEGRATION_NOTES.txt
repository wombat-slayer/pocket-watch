SPRINT 10 INTEGRATION NOTES
============================

## Accounts.jsx — new prop required

In App.jsx, pass the new prop when rendering <Accounts>:

  onUpdateStatementDate={(acctId, date) => setAccounts(as => as.map(a => a.id===acctId ? {...a, lastStatementDate: date} : a))}

Full example in App.jsx Accounts render call:
  <Accounts
    accounts={accounts}
    transactions={transactions}
    netWorthHistory={netWorthHistory}
    recurrences={recurrences}
    onAdd={...}
    onEdit={...}
    onDelete={...}
    onToggleCleared={...}
    onReconcile={...}
    onUpdateStatementDate={(acctId, date) => setAccounts(as => as.map(a => a.id===acctId ? {...a, lastStatementDate: date} : a))}
  />

## migrateData in App.jsx — account field backfill

In the account migration/backfill section of migrateData, you may optionally add:
  lastStatementDate: undefined

This field is NOT required — the Accounts component handles undefined gracefully
(the badge simply does not render when lastStatementDate is absent/undefined).

## Budget Alert native notifications

In handleBudgetAlert (or wherever budget threshold alerts are fired) in App.jsx,
replace any console.warn with the following:

  try {
    const { sendNotification } = await import('@tauri-apps/plugin-notification');
    sendNotification({ title: 'Budget Alert', body: `${category} has reached ${pct}% of its limit.` });
  } catch { console.warn(`[Budget Alert] ${category} ${pct}%`); }

This degrades gracefully — if the Tauri notification plugin is unavailable (e.g. browser
dev mode), it falls back to console.warn without throwing.

The plugin must be registered in src-tauri/Cargo.toml and tauri.conf.json:
  Cargo.toml: tauri-plugin-notification = "2"
  tauri.conf.json plugins section: { "notification": { "all": true } }

## Settings.jsx — props required for Data Health

Settings now uses `accounts` and `budgets` props for the Data Health panel.
These were already passed in the existing signature:
  Settings({ transactions, accounts, budgets, goals, ... })
No additional prop wiring needed.
